About

Agentic India 30 is a 3-day hands-on hackathon designed to introduce first-year and early-year students to the power of AI-driven agents, automation, and real-world problem solving. The event blends learning, building, and collaboration—helping students move from concepts to working prototypes. The hackathon kicks off with a Developer Day focused on fundamentals, tools, and ideation, followed by 48 hours of intense hacking, where teams design intelligent assistants that solve practical student and campus problems. Participants will work in teams of 2–4, guided by mentors, checkpoints, and fun activities, to build useful, beginner-friendly AI applications. The focus is not just on innovation—but on clarity, usability, and learning-by-doing.

Challenges

Campus Buddy Chatbot (RAG Lite)

Build a chatbot that answers college FAQs (departments, clubs, library rules, canteen timings, bus routes) using a small dataset (PDF/CSV). Must-have: upload docs → ask questions → show the source snippet used.

Explain Like I’m New” Study Companion

An assistant that explains any concept (e.g., pointers, Ohm’s law, vectors) in 3 levels: beginner, exam-ready, and real-life analogy. Must-have: user chooses level + gets examples + 3 practice questions

Mini Code Mentor (Debug + Explain)

A web app where a student pastes code and the assistant suggests fixes and explains errors simply. Must-have: handles at least 2 languages (e.g., Python + C) + explains in plain English.

Notes-to-Quiz Generator

Create a tool that converts class notes (text/PDF) into MCQs + short answers with difficulty levels. Must-have: at least 20 questions output + answer key + topic tags.

Placement Prep Starter (1st Year Edition)

Assistant that creates a skill roadmap (coding basics, Git, LinkedIn, communication) over 8 weeks. Must-have: weekly plan + resources list + progress checklist.

Resume Starter for Freshers

A resume builder that takes basic details and generates a one-page resume for a first-year student (projects, skills, clubs). Must-have: ATS-friendly formatting + generates 1 LinkedIn “About” section.

Directory Listing of Mobile Marketing Ecosystem in Asia

Build a consumer driven marketplace referred here as an interactive marketplace since al the players in the system may interact for example the marketer , the channel, the media, , the vendor, and the end user.

It will be a community driven platform which lists the players involved in Mobile Advertising and marketing in Asia , build a digital interactive platform which thrives on listing done by users including the platform and service providers who could list themselves up and could get reviewed by marketers who have used the various services provided by these service providers.

The marketplace will be design to be a multi platform-classified marketplace for end user to learn the various player in the industry /segment and to be able to pull together a solution between them

The platform, built exclusively for MMA would allow users them to understand the full services that a particular vendor provides.

The interactive platform could be a one stop platform for all the marketers and consumers and will allow them to choose the best service providers in the category

This platform will maintain the latest ecosystem map of all the various service provider in the marketplace.

The demonstration of the listing should be in a matrix form with all the details pertaining to it, just a click away.

Participants are expected to come up with an interactive directory listing of these service providers, which will largely help in keeping a track of the marketing industry, the directory must have the interactive mapping of each entity.

The objective of the project is to choose and application that will help conduct an user experience that is easy to use and navigate . it should have a provision to practice primary search, and have a journey map thru the design to explore the entire tool:

  • User experience from start to end should incorporate selection create to add or shortlist any parts of the marketplace to customise ones own ideal solution between various part of the ecosystem.
  • Design thinking to design doing
  • Creating account is mandatory
  • When listing the company, users are required to fill in mandatory fields, the indicator of mandatory fields
  • New Category within the ecosystem can be added by the user , subject to approval by MMA
  • Promote listing as a feature
  • Sorting listing should be category, alphabetic and country wise etc ; filters for sorting to eb provided.
  • Search to be enabled
  • Key focus to be given to:
    • User Interface / In app experience
    • Boosting and promotion
    • Rating system
    • Featured listing
